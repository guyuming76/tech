使用cgdb调试C++应用，例如minidbg

 

安装　cgdb, 我在gentoo 上遇到的问题是默认的版本在　cgdb 模式下按　o 试图打开文件对话框就卡死了：https://github.com/cgdb/cgdb/issues/135　，　解决办法是安装　dev-util/cgdb-9999::gentoo 版本，也就是从　github master branch 下载最新版本，而不是某个branch 或　tag 的版本，打卡　gentoo 的 .ebuild 文件可以看到对　9999 这个版本号的处理．

 

可以用　 cgdb -args ./minidbg ./unwinding　命令启动　cgdb 调试　./minidbg, 并把　./unwinding 作为参数传给 minidbg. 用这种方法可以调试一些程序的初始化，比如这个问题：https://github.com/TartanLlama/minidbg/issues/9

 

 

进入系统后首先在 gdb 窗口输入　b main ，就是在main 函数上设一个断点，　然后　r 开始运行，停到 main 上．

 

如果要在别的文件上设断点，可以 先按　ESC ，进入　cgdb 模式（上半部分代码窗口），然后按　o 进入打开文件对话框，选择需要打开的文件，打开文件后，光标停在目标行，按空格键设置断点

 

 

更多操作方法请查看简明扼要的官方使用指南．

 

Minidbg 调试应用比如unwinding 时，需要用户交互输入调试命令，cgdb 指南介绍有打开另外的输入tty功能，我不确定是否可以用在此处．我的方法是在 Bash 里用　./minidbg ./unwinding启动; 　再使用　htop 找到　minidbg 的进程 pid; 然后单独　sudo cgdb 启动　cgdb 程序，进入　gdb 窗口后用命令　attach pid 附加到minidbg 进程调试．

 

 

 

Attach 到 minidbg 进程后，可以先按　ESC 进入上部cgdb 代码窗口，按o 打开文件窗口，找到并打开代码文件，设好断点，然后按　i 键，切换到下方　gdb 窗口，按　c 继续minidbg 进程运行．接着到minidbg 窗口输入命令比如　step, 让后可以看到　cgdb 进程里断点击中情况．

 

 

有时需要在　gdb 窗口用　p 命令显示变量值，对于一个大的对象，一行显示不完，可以用　gdb 命令　set print pretty on 设置多行显示．

 

最后具体问题参见：　https://github.com/TartanLlama/minidbg/issues/10　　．　另外，cgdb 调试挺方便，但同时我还用　emacs lsp-mode 查看项目代码．
